I want to build a cross-platform Examination Biometric Verification Application
(Android + Windows Desktop + Tablet).

The app workflow and features are as follows:

=====================================
1. OPERATOR LOGIN & REGISTRATION
=====================================

When the app opens, first screen should be Operator Login / Registration.

Operator must fill:
- Operator Selfie (mandatory – capture from device camera)
- Full Name
- Mobile Number (10 digits, validation required)
- Aadhaar Number (12 digits, masked input)
- Optional: Operator ID (auto-generated by system)

After successful registration:
- Save operator details locally (offline support)
- Allow login using mobile number + OTP or PIN
- Show operator profile after login

=====================================
2. EXAM SELECTION & DATA DOWNLOAD
=====================================

After login, operator must:
- Select Exam Name from list
- Select Exam Date
- Select Exam Type:
   1. Mock
   2. Live Exam

IMPORTANT:
- Internet connection is MANDATORY only at the time of data download.
- Data must be downloaded from the central server.

Provide buttons:
- Download Mock Data
- Download Exam Data

Downloaded data includes:
- Candidate list (Application No / Roll No only)
- Candidate name, photo (if available)
- Center code, room/class details

NOTE:
- OMR numbers are NOT pre-mapped.
- OMR mapping will be done only during live registration on exam day.

After download:
- App must work fully offline.
- No internet required during mock or exam process.

=====================================
3. MOCK DAY FUNCTIONALITY
=====================================

On mock day:
- Allow dummy / mock registration
- Test camera and fingerprint devices
- Capture photo & fingerprint
- Scan sample OMR
- Validate device functionality
- Show success/failure logs
- No live server sync on mock day

=====================================
4. EXAM DAY PROCESS (2 ROUNDS)
=====================================

-------------------------------------
ROUND 1 – ATTENDANCE (GATE ENTRY)
-------------------------------------

Steps:
1. Enter Application Number / Roll Number
2. Fetch & display candidate details from local data
3. Verify candidate identity (visual / optional face match)
4. Mark Attendance (Present / Absent)
5. Save attendance locally
6. Prevent duplicate attendance entries

Stored data:
- Application No
- Attendance status
- Date & time stamp
- Operator ID
- Device ID

-------------------------------------
ROUND 2 – ENROLLMENT (CLASSROOM)
-------------------------------------

Steps:
1. Enter Application Number / Roll Number
2. Fetch candidate details
3. Capture Live Candidate Photo
4. Capture Fingerprint
5. Scan OMR Code
6. Map scanned OMR Number with Application / Registration Number
7. Validate duplicate OMR (OMR can be used only once)
8. Mark candidate as Successfully Registered
9. Lock record after successful registration

IMPORTANT:
- OMR number mapping happens ONLY at this stage.
- OMR numbers are NOT pre-assigned before exam.
- One OMR = One candidate only.

Stored data:
- Application No
- Captured photo
- Fingerprint template
- OMR number (mapped live)
- Registration status
- Date & time stamp
- Operator ID
- Room/Class number

=====================================
5. BIOMETRIC & DEVICE SUPPORT
=====================================

- Fingerprint device support (USB / OTG)
- Camera capture (front/back)
- Barcode / QR scanner for OMR
- Auto device detection
- Proper error handling for device failure

=====================================
6. OFFLINE & SYNC MANAGEMENT
=====================================

- Full offline operation after data download
- Encrypted local database storage
- Sync data to server only when internet is available
- Manual and auto sync options
- Sync status:
   Pending / Synced / Failed

=====================================
7. ADMIN & SECURITY
=====================================

- Role-based access (Operator / Supervisor / Admin)
- Data encryption (AES-256)
- Prevent data tampering
- No export without admin authorization
- Auto logout on inactivity
- Block duplicate registration & OMR reuse

=====================================
8. REPORTS & LOGS
=====================================

- Round 1 Attendance Report
- Round 2 Enrollment & OMR Mapping Report
- Operator activity logs
- Device usage logs
- Sync status report

=====================================
9. UI REQUIREMENTS
=====================================

- Fast, minimal UI for exam environment
- Large buttons & clear workflow
- Hindi & English language support
- Clear error, warning & success messages

=====================================
10. TECHNOLOGY STACK (SUGGESTED)
=====================================

- Frontend: Flutter / React Native
- Backend: Node.js / Java Spring Boot
- Offline DB: SQLite
- Server DB: PostgreSQL / MySQL
- Biometric SDK integration
- REST APIs for data download & sync

Generate:
- Complete app architecture
- Screen flow diagrams
- Database schema (with OMR live mapping)
- API request/response samples
- Sample code structure for Android & Windows
